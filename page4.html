OSI MODEL

The Open Systems Interconnection model (OSI model) is a reference model from the International Organization for Standardization (ISO) that "provides a common basis for the coordination of standards development for the purpose of systems interconnection.

In the OSI reference model,the communications between systems are split into seven different abstraction layers: Physical, Data Link, Network, Transport, Session, Presentation, and Application.

7 Layers in OSI Model:

  • Physical Layer
  • Datalink Layer
  • Network Layer
  • Transport Layer
  • Session Layer
  • Presentation Layer
  • Application Layer

Physical Layer

The Physical Layer is the first layer of the OSI model, responsible for the transmission and reception of raw data bits between devices.

It defines the electrical, mechanical, and procedural aspects of data transmission over physical media.

Functions of the Physical Layer:

  • Bit-level transmission over physical media.
  • Physical addressing.
  • Signal encoding and modulation.

Data Link Layer

The Data Link Layer establishes, maintains, and terminates connections between devices.

It ensures reliable transmission of data frames over the physical layer, handling error detection and correction.

Functions of the Data Link Layer:

  • Frame synchronization.
  • Flow control and error control.
  • Media access control.

Network Layer

The Network Layer facilitates communication between devices across different networks.

It provides logical addressing, routing, and forwarding of data packets.

Functions of the Network Layer:

  • Logical addressing (e.g., IP addresses).
  • Routing and forwarding.
  • Fragmentation and reassembly.

Transport Layer

The Transport Layer ensures reliable and orderly delivery of data between devices.

It segments and reassembles data into manageable units, ensuring integrity and flow control.

Functions of the Transport Layer:

  • Segmentation and reassembly.
  • Connection establishment and termination.
  • Flow control and error recovery.

Session Layer

The Session Layer establishes, maintains, and terminates connections (sessions) between applications.

It provides services for session establishment, synchronization, and checkpointing.

Functions of the Session Layer:

  • Session establishment, maintenance, and termination.
  • Dialog control (half-duplex or full-duplex).
  • Synchronization of data exchange.

Presentation Layer

The Presentation Layer ensures that data exchanged between systems is readable and usable by both sender and receiver.

It handles data formatting, encryption, and compression to ensure compatibility between different systems.

Functions of the Presentation Layer:

  • Data translation and encryption.
  • Data compression and decompression.
  • Character code translation (e.g., ASCII to EBCDIC).

Application Layer

The Application Layer provides services directly to user applications.

It enables user-level interaction with the network, supporting various network applications such as email, web browsing, and file transfer.

Functions of the Application Layer:

  • Network virtual terminal.
  • File transfer, access, and management.
  • Email, web browsing, and other network services.
