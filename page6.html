Routing Information Protocol (RIP)

Routing Information Protocol (RIP)

RIP is one of the first routing protocols to be created. It's used in both LANs and WANs and operates on the Application layer of the OSI model.

There are multiple versions of RIP including RIPv1 and RIPv2. RIPv1 determines network paths based on the IP destination and hop count.

RIPv1 interacts with the network by broadcasting its IP table to all routers connected to the network. RIPv2 sends its routing table to a multicast address and offers authentication measures.

Main limitations of RIP include its maximum hop count of 15, lack of scalability, and limited route metric.

Pros:

  • Historical Significance
  • Operational Simplicity
  • Application Layer Operation
  • Multicast Capability (RIPv2)
  • Enhanced Security (RIPv2)

Cons:

  • Maximum Hop Count
  • Lack of Scalability
  • Broadcaster (RIPv1)
  • Limited Route Metric
  • Slower Convergence
Open Shortest Path First (OSPF)

Open Shortest Path First (OSPF)

Open Shortest Path First or OSPF protocol is a link-state IGP that utilizes the Shortest Path First (SPF) algorithm to calculate the most efficient data transmission paths. OSPF routers maintain databases detailing information about the surrounding network topology, gathered from Link State Advertisements (LSAs) sent by other routers.

OSPF employs the Dijkstra algorithm to dynamically adjust to network topology changes, ensuring continuity in data transmission. It also offers protocol change authentication to maintain data security and scalability to large environments.

Pros:

  • Efficient Routing
  • Detailed Network Insight
  • Dynamic Adaptability
  • Security Features
  • Highly Scalable

Cons:

  • Complex Configuration
  • Higher Overhead
  • Sensitive to Topology Changes
  • Resource Intensive
  • Potential for Large LSDB
Border Gateway Protocol (BGP)

Border Gateway Protocol (BGP)

Border Gateway Protocol or BGP is the routing protocol of the internet, classified as a distance path vector protocol. It was designed to replace EGP with a decentralized approach to routing. BGP employs the BGP Best Path Selection Algorithm to choose the best routes for data packet transfers, typically preferring routes with the shortest path to the destination.

BGP allows administrators to customize routing decisions based on various factors like weight, local preference, AS_Path length, and more. It only sends updated router table data when changes occur, requiring manual configuration for topology changes. Additionally, BGP provides authentication measures to ensure secure data exchange between approved routers.

Pros:

  • Internet Backbone
  • Decentralized Design
  • Customizable Path Selection
  • Efficient Updates
  • Granular Routing Decisions
  • Authentication

Cons:

  • Complex Configuration
  • Potential Instability
  • Scalability Concerns
  • Vulnerabilities
  • Learning Curve
  • Convergence Time
Interior Gateway Protocol (IGRP)

Interior Gateway Protocol (IGRP)

Interior Gateway Protocol or IGRP is a distance vector routing protocol produced by Cisco. It was designed to address the limitations of RIP and function more effectively within larger connected networks.

IGRP uses metrics such as bandwidth, delay, reliability, and load to compare the viability of routes within the network. However, only bandwidth and delay are used under IGRP’s default settings.

IGRP is ideal for larger networks because it broadcasts updates every 90 seconds and has a maximum hop count of 255. This allows it to sustain larger networks than a protocol like RIP. IGRP is also resistant to routing loops because it updates itself automatically when route changes occur within the network.

Pros:

  • Enhanced Scalability
  • Multiple Metrics
  • Frequent Updates
  • Loop Resistance
  • Cisco Legacy

Cons:

  • Proprietary Protocol
  • Limited Default Metrics
  • Superseded by EIGRP
  • Larger Overhead
  • Potential Complexity
Exterior Gateway Protocol (EGP)

Exterior Gateway Protocol (EGP)

Exterior Gateway Protocol or EGP is a protocol that is used to exchange data between gateway hosts that neighbor each other within autonomous systems. In other words, EGP provides a forum for routers to share information across different domains. The most high profile example of an EGP is the internet itself. The routing table of the EGP protocol includes known routers, route costs, and network addresses of neighboring devices. EGP was widely-used by larger organizations but has since been replaced by BGP.

The reason why this protocol has fallen out of favor is that it doesn’t support multipath networking environments. The EGP protocol works by keeping a database of nearby networks and the routing paths it could take to reach them. This route information is sent on to connected routers. Once it arrives, the devices can update their routing tables and undertake more informed path selection throughout the network.

Pros:

  • Data Exchange Between Autonomous Systems: Allows gateway hosts to share information across distinct network domains, effectively acting as a bridge.
  • Foundation of Early Internet: Served as a precursor and essential component to the modern internet’s formation.
  • Routing Database: Contains comprehensive information, including known routers, route costs, and the addresses of neighboring devices.
  • Path Information Sharing: Sends route data to neighboring routers, helping them update their tables and make better routing decisions.

Cons:

  • Lack of Multipath Support: EGP isn’t suitable for modern multipath networking environments, limiting its adaptability.
  • Obsolete: Has been largely phased out in favor of more advanced protocols, notably BGP.
  • Limited Scalability: As networks grew, EGP struggled with handling larger and more intricate systems.
  • Static Path Determination: While EGP keeps a database of nearby networks, its path determinations are more static, making it less flexible than newer protocols.
  • Potential for Redundancy: EGP’s method of sharing all route data with neighboring routers can lead to redundant data transmission and larger routing tables.

Routing Protocol Classification

Category Description Protocols
Distance Vector Routing protocols that determine the best path to a destination based on distance or hop count. RIP, IGRP, EIGRP
Link State Routing protocols that maintain a detailed view of the entire network topology. OSPF, IS-IS
Interior Gateway Protocol (IGP) Routing protocols used within an autonomous system (AS). RIP, IGRP, EIGRP, OSPF, IS-IS
Exterior Gateway Protocol (EGP) Routing protocols used between autonomous systems. BGP
Classful Routing protocols that do not carry subnet mask information. RIP v1, IGRP
Classless Routing protocols that carry subnet mask information. RIP v2, EIGRP, OSPF, BGP

IPV4 & IPV6 ADDRESS

IPv4

  • ✔ Address length: 32 bits
  • ✔ Notation: Decimal format separated by periods (e.g., 192.168.0.1)
  • ✔ Address space: Approximately 4.3 billion addresses
  • ✔ Address Configuration: Manual or DHCP
  • ✔ Header size: Fixed 20 bytes, variable options
  • ✔ Security: IPsec optional
  • ✔ Special addresses: Private addresses reserved for local networks
  • ✔ Migration challenges: Address exhaustion, NAT necessity

IPv6

  • ✔ Address length: 128 bits
  • ✔ Notation: Hexadecimal format separated by colons (e.g., 2001:0db8:85a3:0000:0000:)
  • ✔ Address space: Huge, approximately 340 undecillion addresses
  • ✔ Address Configuration: Stateless autoconfiguration or DHCPv6
  • ✔ Header size: Fixed 40 bytes, simpler and more efficient
  • ✔ Security: Integrated security with IPSec mandatory
  • ✔ Special addresses: No need for private addresses due to vast address space
  • ✔ Migration challenges: Compatibility issues with legacy systems, transition mechanisms required
